Seamless API Specification

Developer documentation for B2B gaming platform integration and dynamic partner routing.

Architecture Note: This module handles outbound HTTP client communication to external B2B partner endpoints. Routing destinations and secrets are resolved dynamically at runtime based on the partnerId.

🔗 Platform Link Generation Engine

To safely pass the user context directly into target game frames, assemble standard launch URLs matching the architecture configuration below:

https://mtwtech.onrender.com/platform?sessionId={sessionId}&partnerId={partnerId

Lifecycle Workflow:

1. Your front-end or affiliate network maps the parameterized link pattern.

2. The incoming router intercepts session and partner properties.

3. Parameters feed straight into the validateSession(sessionId, partnerId) execution loop before full game rendering asset hooks are authorized.

validateSession Internal Function

Verifies the player's session token and performs auto-login routines when loading or initializing a game.

POST ${integration.url}/validate

Function Arguments

  • sessionId string — The unique session token passed by the user client.
  • partnerId string — The identifier used to resolve specific B2B partner configuration.

Outgoing Payload to Partner

{
  "token": "string (sessionId)",
  "secret": "string (B2B partner secret key)"
}

Expected Response

{
  "username": "Player_1",
  "balance": 5000
}

debit Internal Function

Triggers a real-time balance debit query whenever a player places a wager inside an active game session.

POST ${integration.url}/debit

Function Arguments

(username, partnerId, sessionId, amount, gameName, roundId)

Outgoing Payload to Partner

{
  "username": "string",
  "token": "string (sessionId)",
  "amount": "number (explicitly cast)",
  "game": "string (Game Title)",
  "roundId": "string (Unique Game Round ID)",
  "secret": "string (B2B partner secret key)"
}

Expected Response

{
  "balance": 4900
}

credit Internal Function

Triggers a credit event to add payout balances, round winnings, or promotional funds back to the player account.

POST ${integration.url}/credit
Internal Transaction Tagging: The system automatically logs metadata types based on the gameName string:
• If it contains "Affiliate" ➜ Tagged as AFFILIATE
• If it contains "Promo", "Cashback", "Quest", "VIP" ➜ Tagged as BONUS_CASH
• Fallback default ➜ Tagged as CREDIT

Function Arguments

(username, partnerId, sessionId, amount, gameName, roundId)

Outgoing Payload to Partner

{
  "username": "string",
  "token": "string (sessionId)",
  "amount": "number (explicitly cast)",
  "game": "string (Game Title)",
  "roundId": "string (Unique Game Round ID)",
  "secret": "string (B2B partner secret key)"
}

Expected Response

{
  "balance": 5200
}
